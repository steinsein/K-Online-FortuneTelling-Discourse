# 코드북 (Codebook)

본 문서는 데이터 처리 및 분류 과정에서 사용된 코딩 체계를 설명합니다.

---

## 1. 유효성 등급 (Validity)

`validator.py`에서 부여하는 게시물 관련성 등급입니다.

| 등급 | 라벨 | 기준 |
|------|------|------|
| V0 | invalid | 운세 서비스와 무관한 게시물 |
| V1 | normal | 운세 키워드 단독 출현 (일반적 관련성) |
| V2 | strong | 운세 키워드 + 동시출현 키워드 확인 (강한 관련성) |

동시출현 키워드 예시: 사주 + (팔자, 풀이, 보다, 상담 등), 타로 + (리딩, 카드, 상담 등)

---

## 2. 광고 분류 (`is_ad`)

| 값 | 설명 |
|----|------|
| `True` | 광고성 콘텐츠 (원고료, 협찬, 체험단, 상담문의 유도 등) |
| `False` | 비광고 콘텐츠 |

주요 광고 키워드: 원고료, 협찬, 체험단, 쿠팡 파트너스, 상담문의, 예약신청 등

---

## 3. 맥락 유형 (`context_type`)

게시물의 주된 맥락을 분류합니다.

| 값 | 설명 | 키워드 예시 |
|----|------|------------|
| 여행 | 여행 중 운세 체험 | 여행, 일본, 도쿄, 대만, 관광 |
| 술집 | 술자리/유흥 맥락 | 술집, 포차, 이자카야, 클럽 |
| 일상 | 일상 기록 내 운세 언급 | 일기, 일상, 맛집, 데이트 |
| 운세집중 | 운세 서비스가 주요 주제 | (위 3개에 해당하지 않는 경우) |

---

## 4. 운세 서비스 채널 유형 (`channel_type`)

사용자가 이용한 운세 서비스의 채널을 분류합니다.

| 값 | 설명 | 키워드 예시 |
|----|------|------------|
| 모바일앱(점신/포스텔러 등) | 운세 전용 모바일 앱 | 점신, 포스텔러, 백점, 운세의신 |
| AI(ChatGPT/챗봇 등) | AI 기반 운세 서비스 | GPT, 지피티, 뤼튼, 클로드, 헬로우봇, 판밍밍 |
| 원격/온라인 상담 | 비대면 개인 상담 | 온라인, 카톡, 전화상담, 화상, 비대면 |
| 상담플랫폼(사주나루/홍카페 등) | 전문 상담 중개 플랫폼 | 사주나루, 홍카페, 천을귀인, 포스텔러(상담) |
| 포털/금융/생활앱 | 포털·금융앱 내 운세 서비스 | 네이버 운세, 토스, 신한생명, 카카오운세 |
| 유튜브/영상 | 유튜브 운세 콘텐츠 | 유튜브, 유튜버, 영상, 구독, 쇼츠 |
| 기타어플 | 기타 앱/서비스 | 어플, 앱, 설치, 결제 |
| 오프라인/방문 | 대면 방문 상담 | 직접 가, 방문, 다녀, 철학관, 점집 |

---

## 5. 사용자 의도 (`user_intent`)

게시물 작성자의 의도를 분류합니다.

| 값 | 설명 |
|----|------|
| 직접체험후기 | 본인이 직접 이용한 서비스에 대한 후기 |
| 정보탐색 | 운세 서비스에 대한 정보를 구하는 게시물 |
| 자가분석/이론토론 | 사주 이론, 해석 방법 등에 대한 토론 |
| 간접후기/전해들은이야기 | 타인의 경험을 전달하는 게시물 |

---

## 6. 채널 이용 태그 (불리언)

`platform_usage_tagger.py`에서 부여하는 불리언 태그입니다.

| 컬럼 | 설명 | 탐지 방식 |
|------|------|-----------|
| `is_app_used` | 운세 전용 앱 이용 여부 | 브랜드명 직접 매칭 또는 (일반명사 + 맥락 키워드) 동시출현 |
| `is_youtube_used` | 유튜브 운세 콘텐츠 이용 여부 | (유튜브/유투브/유튜버) + (사주/타로/운세) 동시출현 |
| `is_ai_used` | AI 운세 서비스 이용 여부 | (GPT/지피티/AI) + (사주/타로/운세) 동시출현 |
